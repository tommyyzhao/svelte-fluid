## What needs attention next

### Priority: Low-level optimizations

1. **Merge 3 velocity masks into 1** ‚Äî currently `applyMask(velocity)` is called after vorticity, gradient-subtract, and velocity advection. Masking once after final advection would cut 3 blits to 1. The intermediate leakage is imperceptible within a single frame. See ADR 0022.

2. **Pre-baked mask texture** ‚Äî compute the SDF mask into a texture once when shape changes, then sample it in `applyMask` instead of recomputing per-fragment. Replaces ~10 ALU ops/frag with a texture fetch. Requires extra FBO + re-bake on shape change.

3. **Stencil-based masking** ‚Äî use stencil buffer instead of full-screen multiply. Only beneficial when masked area is large (thin annuli, small circles).

4. **`buildSnippet` doesn't emit `containerShape`** ‚Äî the playground's "Copy as code" generates a `<Fluid>` snippet missing the container shape prop entirely. Needs synthesis of the compound object from scalar state vars.

### Other items

5. **Playground shape preview UX** ‚Äî outer boundary at default `outerHalfW=0.5` draws at canvas edge (invisible against border). Could skip drawing outer when it matches defaults.

6. **`hdrMultiplier` ignores aspect ratio** ‚Äî circle/annulus area fraction uses `œÄ*r¬≤` but `r` is height-normalized; on wide canvases the UV-space area is actually `œÄ*r¬≤/aspect`. Pre-existing issue.

7. **npm publish** ‚Äî package for public consumption once API surface is stable. The `cornerRadius` ‚Üí `innerCornerRadius` breaking change should be in the first published version, not a post-publish break.
