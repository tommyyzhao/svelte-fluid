### 3. autoPause + context loss recovery (commit `bbfb787`)

Root cause of all-preset crashes: browsers silently cull WebGL contexts under GPU
pressure, and the engine had no `webglcontextlost`/`webglcontextrestored` handlers.

**Three layers of defense added:**

| Layer | Trigger | Action |
|-------|---------|--------|
| `autoPause` (IntersectionObserver) | Canvas scrolls out of viewport | `engine.pause()` — stops RAF, keeps GL context |
| `autoPause` (Page Visibility API) | Tab hidden / minimized | `engine.pause()` — stops RAF |
| Context loss handler | Browser culls GL context | Stops RAF, flags `contextLost`, full reinit on restore |

- New `autoPause` prop on `<Fluid />` (default `true`)
- New public API: `pause()`, `resume()`, `isPaused` on `FluidHandle` + `FluidEngine`
- `webglcontextlost`: `e.preventDefault()`, stop RAF, set flag
- `webglcontextrestored`: full reinit (shaders, programs, FBOs, dithering, splats), resume
- All GL-calling methods (`splat`, `setConfig`, `update`) guard against `contextLost`
- Updated all 7 preset components with new FluidHandle methods

## Architecture state after session

### Shape system
- `ContainerShape` union: `circle` | `frame`
- `applyMaskShader`: branches on `uShapeType` uniform (0/1)
- Display shader `CONTAINER_MASK`: same branching
- `container-shapes.ts`: pure TS SDF functions mirror GLSL for testing
- Adding new shapes: extend union, add SDF branch in mask shader + display shader

### Visibility / lifecycle
- `autoPause` (default true): IO pauses RAF on scroll-out, Page Visibility pauses on tab hide
- `lazy` (default false): full teardown/rebuild on scroll in/out (frees GL context slot)
- Context loss/restore: automatic reinit, transparent to consumer
- `pause()` / `resume()` / `isPaused`: public API for manual control

### Medium-term
5. **Rounded-rect shape** — natural next ContainerShape variant. Needs a proper
   rounded box SDF (Inigo Quilez formula) with corner radius uniform. Would
   validate the shape extensibility pattern for a third time.
6. **Scale HDR multiplier by container area** — the 10x HDR multiplier on random
   splat colors causes oversaturation in confined containers. A general fix would
   scale the multiplier by `containerArea / canvasArea`.
7. **Memory profiling** — use Chrome DevTools to verify no leaked GL resources
   after rapid resize cycles and lazy teardown/rebuild.

### Longer-term
8. **Shared WebGL context** — single context rendering multiple viewports would
   eliminate the context limit entirely. Major architectural change.
9. **Simulation state serialization** — save/restore fluid state across resize
   or context loss (currently the simulation resets).
10. **Performance optimization** — profile the step() loop at higher sim resolutions
    (256, 512). The 4x applyMask blit per frame adds up at high res. Consider
    batching mask applications or using stencil-based approaches for perf-critical paths.
